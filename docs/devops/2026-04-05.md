# DevOps & CI/CD Audit — 2026-04-05

## Current State

- **E2E:** Maestro (local only, auth flows)
- **Linting:** ESLint v9 + Prettier + TypeScript strict
- **Pre-commit:** Husky (full lint suite, no lint-staged)
- **Deploys:** EAS OTA updates (manual trigger via `npm run publish`)
- **CI/CD pipeline:** None
- **Unit tests:** None
- **Coverage tracking:** None
- **eas.json:** Missing

---

## Recommendations (Priority Order)

### 1. GitHub Actions — PR Validation

Automate lint, type-check, and build validation on every push/PR. Catches regressions before review.

**Workflow triggers:** push to main, all PRs.
**Jobs:** eslint, prettier check, tsc --noEmit, expo doctor.

### 2. lint-staged

Pre-commit hook currently lints the entire codebase. With lint-staged, only staged files are checked — commits go from seconds-to-minutes down to sub-second.

```json
"lint-staged": {
  "*.{ts,tsx}": ["eslint --fix", "prettier --write"]
}
```

### 3. Jest + React Native Testing Library

Zero unit tests today. Highest-ROI addition after CI.

- `jest-expo` preset (Expo's official Jest integration)
- `@testing-library/react-native` for component behavior testing
- Target: custom hooks, utility functions, complex components
- Colocate tests: `Component.test.tsx` next to `Component.tsx`

### 4. EAS Build Profiles (eas.json)

Define build profiles for proper pipeline:

- **development** — internal distribution, debug builds for simulators
- **preview** — internal distribution for TestFlight/testers on PR branches
- **production** — App Store / Play Store submissions

Enables automated preview builds on PRs (reviewers get install links).

### 5. commitlint

Enforce conventional commit format automatically via commit-msg hook. `@commitlint/config-conventional` rejects malformed messages.

### 6. EAS Auto-Deploy on Merge

Trigger `eas update` automatically when PRs merge to main. Removes manual `npm run publish` step.

### 7. Code Coverage (Codecov)

Once unit tests exist:

- Jest `--coverage` with `coverageThreshold` in config
- Codecov GitHub integration posts coverage diffs on PRs
- Block merges if coverage drops

### 8. Dependabot / Renovate

Auto-create PRs for dependency updates with changelogs. Combined with CI, patch bumps that pass validation can be auto-merged.

### 9. Playwright Web E2E

Cover the web platform in CI:

- Auth flows, navigation, form submissions
- Visual regression screenshots
- Runs headless in CI — no device costs

### 10. Bundle Size & Security Scanning

- `expo-bundle-analyzer` or webpack-bundle-analyzer for web
- `npm audit` in CI (fail on high/critical)
- GitHub CodeQL for static analysis (free Action)
- Track bundle size in CI, alert on significant increases

---

## PR Preview Builds (EAS + GitHub)

EAS supports automatic preview builds triggered by GitHub PRs via `expo-github-action`. Reviewers can install and test the actual build from the PR — catches UI regressions that linting and type-checking cannot.
