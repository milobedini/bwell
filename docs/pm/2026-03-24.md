## PM Brief — 2026-03-24

### Current State

The app has a solid foundation: auth, role-based access, therapist-patient relationships, assignment lifecycle, questionnaire engine (PHQ-9, GAD-7, PDSS), a polished Activity Diary with auto-save, and a therapist timeline with filtering. The core plumbing — assign homework, complete it, review it — works for questionnaires and the Activity Diary. However, the product is still far from delivering a complete Depression CBT program end-to-end. Only 2 of the ~10 clinical tools defined in the proposal are built, and the home screens for both patient and therapist are placeholder-quality.

### Unfinished Work (Fix First)

- [ ] **TODO in AttemptPresenter: psychoeducation and exercise presenters missing** (`components/attempts/presenters/AttemptPresenter.tsx:27`) — These module types exist in the BE schema but hitting them in the app shows a dead-end "Not available yet" screen. If a therapist assigns a psychoeducation or exercise module, the patient sees an empty state with no explanation. This is a broken experience for any program that includes those module types.

- [ ] **Patient home screen is a single button** (`components/home/PatientHome.tsx`) — The patient lands on a screen with one button ("Active attempts"). There is no dashboard, no at-a-glance view of upcoming assignments, due dates, or progress. For a patient opening the app between sessions to do their homework, this is a poor first impression. The home screen should surface what they need to do this week.

- [ ] **Therapist home screen is two buttons** (`components/home/VerifiedTherapistHome.tsx`) — Similarly sparse. A therapist needs to quickly see which patients have submitted homework this week and which are falling behind. Right now they have to navigate through multiple screens to find this information.

- [ ] **No pull-to-refresh on most lists** — Only `TherapistLatestAttempts` has pull-to-refresh. Patient attempts, assignments, programs, and modules lists all lack it. On mobile, pull-to-refresh is the expected way to get fresh data. Its absence makes the app feel unresponsive.

- [ ] **No empty states on home screens** — The home components (`PatientHome`, `VerifiedTherapistHome`) have no handling for when a patient has zero assignments or a therapist has zero clients. New users see content that leads nowhere.

- [ ] **Hardcoded shadow color in assignments tab** (`app/(main)/(tabs)/assignments/index.tsx:93`) — `shadowColor: '#34715D'` should use `Colors` tokens per project conventions.

### MVP Status

**MVP is partially defined but not yet achieved.** Using the MVP criteria:

| Criterion | Status |
|-----------|--------|
| Can a therapist onboard and get verified? | Done |
| Can a therapist assign work to a patient? | Done |
| Can a patient complete the assigned work? | Partial — works for questionnaires and Activity Diary only |
| Can a therapist review what the patient did? | Done (timeline with filters) |
| Does it cover at least one complete program (e.g., Depression)? | **No** — Depression requires ~10 tools; only 2 are built |

The MVP gap is clear: **the app cannot deliver a single complete CBT program.** A therapist running a Depression program needs, at minimum, the Activity Diary, 5 Areas Model, General Goals, and Weekly Goals to cover the first half of treatment. Thinking Traps and Thought Challenging come later in a typical Depression program but are essential for the full course.

### MoSCoW Prioritisation

**Must Have (MVP — required to run a Depression program end-to-end)**

- **5 Areas Model** — The foundational CBT tool. Every program uses it. The therapist explicitly flagged this as a core component for all three tiers (self-help, CBT-guided, PWP-guided). Without it, the app is missing the central clinical framework. Patients need to map Situation → Thoughts → Emotions → Behaviours → Physical symptoms, then reflect on how to break the cycle.

- **General Goals** — Therapy cannot proceed without goals. The therapist needs to set goals with the patient in session 1-2, rate them (0-10), and re-rate at midpoint and end. This is standard clinical practice and the proposal explicitly requires mid-therapy re-rating prompts.

- **Weekly Goals** — Bridges the gap between sessions. Patients need a structured way to plan what they will do each week, tie it to their Activity Diary, and tick things off for a sense of achievement. The proposal ties this directly to engagement and homework compliance.

- **Patient home dashboard** — Not a "feature" in the proposal, but a blocker for usability. A patient needs to open the app and immediately see: what is due this week, what they have completed, and what is overdue. Without this, the app fails the basic "open and know what to do" test.

- **Therapist home dashboard** — Same reasoning. A therapist checking between sessions needs at-a-glance: which patients submitted this week, which have overdue work, any concerning questionnaire scores.

**Should Have (Post-MVP, High Priority — needed for complete Depression program)**

- **Thinking Traps** — Introduced mid-way through a Depression program. Patients learn to identify cognitive distortions. High clinical value but therapists can work around its absence for the first few sessions by using paper worksheets.

- **Thought Monitoring** — Logs unhelpful thoughts and tags them with thinking traps. Builds on Thinking Traps. Important for the cognitive phase of CBT but not session-1 material.

- **Thought Challenging** — The structured evidence-for/against worksheet. This is the payoff of the cognitive work — patients learn to challenge their automatic thoughts. High value but depends on Thinking Traps and Thought Monitoring being built first.

- **Psychoeducation presenter** — The module type exists in the BE but has no FE presenter. Needed for Panic Disorder (fight-or-flight education) and will be needed for every program's introductory material. Lower complexity than clinical tools.

**Could Have (Future — expands breadth but not depth)**

- **Longitudinal Model / Extended CBT Cycle** — Beck formulation. Used once or twice per therapy, not weekly homework. High clinical value but low frequency of use. Can be done on paper initially.

- **Belief Change Evidence Table** — Advanced cognitive tool used in later sessions. Depends on Thought Challenging being well-established first.

- **Modifying Unhelpful Beliefs** — Structured worksheet for belief modification. Same dependency chain as above.

- **Exercise presenter** — Module type exists in BE with no FE UI. Lower priority than psychoeducation because exercises are typically simpler content.

- **Self-help program recommendation** — Suggesting programs based on questionnaire scores. Nice for the self-help tier but the priority tier is therapist-guided where the therapist chooses the program.

- **GAD, Panic Disorder, OCD programs** — Expanding to other programs. Must not happen until the Depression program is fully complete and proven.

**Won't Have (This Cycle)**

- **PTSD, Agoraphobia, Social Anxiety, Phobias programs** — These are listed in the proposal but each requires specialist clinical tools beyond the core CBT toolkit. Defer until the core depression/anxiety programs are solid.

- **ACT (Acceptance and Commitment Therapy)** — A different therapeutic modality entirely. The proposal marks it with a question mark. Out of scope.

- **Sleep, Assertiveness programs** — Labelled "additional" in the proposal. Not core CBT.

- **Self-help tier** — The proposal lists self-help as tier 1, but the business partner explicitly said tier 2 (CBT therapist-guided) should be developed first. Self-help also requires program recommendation logic that does not exist yet.

### Recommendation

**1. Polish the existing experience first (1-2 days)**

Before building any new clinical tool, fix the rough edges:
- Build a proper Patient home dashboard showing upcoming/overdue assignments and recent scores
- Build a Therapist home dashboard showing client submission status at a glance
- Add pull-to-refresh to all list screens (patient attempts, assignments, programs)
- Add empty states to home screens
- Fix hardcoded hex values
- This is not optional — shipping a new CBT tool into an app where the home screen is a single button undermines the entire product

**2. Build the 5 Areas Model (next feature)**

This is the highest-value, highest-impact tool to build next. Rationale:
- It is used across all three tiers (self-help, CBT, PWP) and all programs
- It is the foundational clinical framework that everything else builds on
- A therapist cannot run a proper Depression program without it
- The proposal describes clear UX requirements: fill in Situation, Thoughts, Emotions, Behaviours, Physical; then reflect on how to break the cycle
- It is self-contained — no dependency on other unbuilt tools

**3. Then General Goals + Weekly Goals (following feature)**

These two go together because they define the structure of therapy (what are we working toward?) and the weekly cadence (what are you doing this week?). Weekly Goals ties directly into the Activity Diary, creating a cohesive homework loop: set goals → plan in diary → complete → reflect.

### Scope Creep Watch

- **Do not build Thinking Traps or Thought Challenging yet.** They are tempting because they are intellectually interesting, but they are mid-therapy tools. A Depression program does not introduce them until sessions 4-6. Build the session 1-3 tools first (5 Areas, Goals).
- **Do not expand to other programs (GAD, Panic).** The Depression program is not complete. Prove the model works for one program before replicating.
- **Do not build the self-help tier.** The business partner said CBT-guided is the priority. Self-help requires additional logic (program recommendations, more psychoeducation) that is not needed for the guided tier.
- **Do not refactor the module type system or shared types architecture.** It works. The current `ModuleType` union and presenter routing pattern are clean. Extend it, do not redesign it.
- **Do not build a notification system.** It will be needed eventually (assignment reminders, due date alerts) but it is infrastructure that does not deliver clinical value right now. The therapist assigns homework in session — the patient knows what to do.
