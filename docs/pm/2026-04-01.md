## MoSCoW Prioritisation

### Must Have (MVP)

- **5 Areas Model** — The foundational CBT framework used in session 1 across every program and every tier. Patient maps a situation across 5 areas (Situation, Thoughts, Emotions, Behaviours, Physical symptoms), then reflects on how to break the cycle. The proposal specifies this for self-help, CBT-guided, and PWP-guided tiers. This is the single most important unbuilt feature. **Four consecutive briefs have recommended it.**
- **General Goals** — Therapy goal-setting with 0-10 ratings at start, mid, and end of therapy. Used from session 1. Multiple goals supported. Reflection prompts specified in proposal.
- **Weekly Goals** — Weekly to-do list tied to the Activity Diary. Patients set behavioural goals, tick them off, reflect with prompt questions. Creates the homework compliance loop that makes guided CBT work.

### Should Have (Post-MVP, High Priority)

- **Thinking Traps** — Learn and identify cognitive distortions. Sessions 4-6 of Depression program. High clinical value but not needed for early sessions.
- **Thought Monitoring** — Log unhelpful thoughts and tag with thinking traps. Depends on Thinking Traps.
- **Thought Challenging** — Structured evidence for/against worksheet. Core cognitive restructuring tool. Depends on Thought Monitoring.
- **Password change / account settings** — Basic account management. Not blocking therapy workflow but expected by users.

### Could Have (Future)

- **Longitudinal Model** — Beck formulation. Advanced, used once or twice per therapy course.
- **Belief Change Evidence Table** — Late-stage cognitive work. Depends on Thought Challenging.
- **Modifying Unhelpful Beliefs** — Structured worksheet. Same dependency chain.
- **Self-help program recommendation** — Suggest programs from questionnaire scores. Valuable for self-help tier.
- **Additional programs** (GAD, Panic, OCD, etc.) — Depression must work completely first.
- **Recurring assignment automation** — BE has recurrence data model but no scheduler.
- **Therapist notes on submissions** — therapistNote field exists on attempts but no UI to write/read them.

### Won't Have (This Cycle)

- **PWP tier differentiation** — Shares tools with CBT tier. Defer until CBT tier is complete.
- **Self-help tier** — Business partners prioritised therapist-guided.
- **ACT, Sleep, Assertiveness programs** — Secondary programs.
- **PTSD, Agoraphobia, Social Anxiety** — Require specialist adaptations.
- **Notification system** — Infrastructure that doesn't deliver clinical value yet.
- **Admin content-authoring UI** — Seed scripts work for now.

## Recommendation

### 1. Build the 5 Areas Model (top priority — no more excuses)

This is the **fourth consecutive PM brief** recommending this. The structural refactors are done. The polish cycle is done. The unified practice model is done. There is nothing left to refactor or improve before building this tool.

What it needs:
- **BE:** New module type `five_areas_model`, Mongoose schema for 5-areas entries (situation, thoughts, emotions, behaviours, physical, reflection), controller for CRUD
- **Shared types:** `FiveAreasEntry` type, add to `ModuleType` enum
- **FE:** Presenter component in `components/attempts/presenters/five-areas/`, form with 5 text areas + reflection prompt, save-as-you-go (like activity diary)
- **Seed:** Add 5 Areas Model module to the Depression program

The proposal is clear on requirements:
- Fill in 5 areas using a real situation where mood dropped
- Each area gets its own text input (Situation, Thoughts, Emotions, Behaviours, Physical)
- Reflection: "How could you break this cycle to improve your mood?"
- Multiple entries supported (one per situation)
- Therapist can review entries

### 2. Build General Goals immediately after

Together with the 5 Areas Model, this covers sessions 1-2 of a Depression program:
- Goal text + current rating (0-10) + mid-therapy rating + end-of-therapy rating
- Multiple goals supported
- Reflection prompts from proposal ("Are these goals in line with your values?", "Are they achievable?", "Are they relevant to improving your mood?")
- 6-week reminder to re-rate (can be a simple prompt, not a notification system)

### 3. Build Weekly Goals to complete the session 1-3 toolkit

After 5 Areas and General Goals, Weekly Goals completes the minimum viable therapy session toolkit. Together with the existing PHQ-9, Activity Diary, and reading content, a therapist could run a real Depression CBT program through sessions 1-3.

### What NOT to build yet

- **Thinking Traps / Thought tools** — Sessions 4-6. Build after the session 1-3 toolkit works.
- **New programs** — Depression must work end-to-end first.
- **Profile/settings** — Not blocking therapy.
- **More structural refactors** — The practice model is done. Build clinical tools on top of it.
- **Therapist notes UI** — The data model supports it but it's not needed for MVP.
